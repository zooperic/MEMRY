What is MEMRY

MEMRY is a handmade e-ink fridge magnet shaped like a Polaroid photo. It connects to WiFi and silently updates to show whatever photo is currently "published" on its paired web account.

Friends and family upload photos through a simple web app. The magnet wakes up every few hours, checks for a new image, renders it on the e-ink display, and goes back to sleep — drawing almost no power in between.

The magic moment: Jo wakes up in the morning. There's a new photo on her fridge — sent last night by a friend in another city. She didn't touch anything. The fridge just remembered.

Why we built it

This started as a birthday gift for Jo (July 18, 2025). The goal was something that felt alive — not a static photo frame, not a digital screen, but something quiet and warm that occasionally surprises you.

The name MEMRY is a deliberate misspelling of "memory." Short, typeable, a little broken — like a polaroid left in the sun.

VidaBay showed this category works. MEMRY is built for the use case VidaBay can't serve: people who want to send moments to someone they love, remotely, without being in the room.

Charging

Plug any USB-C cable into the XIAO's USB-C port (accessible via shell edge notch). XIAO charges the LiPo at up to 500mA. A depleted 300mAh battery charges fully in about 40 minutes.

XIAO charging LED: Orange/red = charging. Off or green = full. (Varies slightly by XIAO batch.)

08

Assembly

Always test electronics on breadboard first. Flash firmware, confirm WiFi connects, confirm display renders. Only then proceed to shell assembly.

Wire display + XIAO on breadboard

Connect per wiring diagram. Temporarily power via USB-C. Flash firmware. Upload test image. Confirm display renders correctly.

Test WiFi + server fetch

Upload a test BMP to your server. Confirm the device fetches and renders it. Measure sleep current if possible (optional but useful).

Fit display into front cavity

Place display face-down into front bay of 3D printed shell. Active area faces the window cutout. PCB faces inward.

Route FPC ribbon

Thread the flat ribbon cable through the FPC channel into the component bay. Don't kink it — e-ink ribbons are fragile at tight bends.

Seat XIAO in component bay

Orient so USB-C port aligns with the notch on the shell edge. Connect jumper wires from FPC breakout to XIAO per pin diagram.

Connect LiPo

Plug JST connector into XIAO's BAT port. Check polarity first. Red = positive. Lay LiPo flat beside XIAO. Nothing should be stacked on top — keep it flat.

Charge before sealing

Plug USB-C and charge fully. Confirm charging indicator. Do a final test — upload photo, watch it refresh. Everything working? Proceed.

Epoxy magnets

Mix 2-part epoxy, fill magnet pockets, press N52 magnets in. Check fridge-facing polarity first — all 4 magnets should attract the fridge, not repel. Cure 30 minutes.

Close back shell

Snap or screw closed. The shell is now sealed. Write device ID inside with a marker before closing if not embossed.

Stick to fridge. Done.

Place on any ferromagnetic surface. WiFi will auto-connect on next wake cycle (within 4 hours). First photo should appear by morning.

Firmware

Written in Arduino C++ for the XIAO ESP32C3. Uses GxEPD2 for display driving, HTTPClient for image fetch, built-in WiFi library.

Flash config (per device, one-time)

// Stored in XIAO flash — set once per unit
const char* WIFI_SSID   = "Jo_Home_WiFi";
const char* WIFI_PASS   = "password123";
const char* DEVICE_ID   = "memry-001";
const char* SERVER_URL  = "https://api.memry.app";
uint64_t    SLEEP_US    = 4ULL * 3600 * 1000000; // 4 hours

Main loop logic

void setup() {
  // 1. Connect WiFi (timeout 20s, retry 3×)
  WiFi.begin(WIFI_SSID, WIFI_PASS);
  
  // 2. Fetch current image from server
  String url = String(SERVER_URL) + "/api/device/" + DEVICE_ID + "/image";
  http.begin(url);
  http.addHeader("If-None-Match", lastEtag); // skip if unchanged
  int code = http.GET();
  
  // 3. If new image (200 OK), render to display
  if (code == 200) {
    display.init();
    // stream BMP from response to display
    renderBMP(http.getStreamPtr());
    display.hibernate(); // CRITICAL — always hibernate after refresh
    storeEtag(http.header("ETag")); // cache in RTC memory
  }
  
  // 4. Deep sleep until next poll
  WiFi.disconnect(true);
  esp_sleep_enable_timer_wakeup(SLEEP_US);
  esp_deep_sleep_start(); // wakes, runs setup() again
}

void loop() {} // never runs — always in setup → sleep cycle

Image processing pipeline

Photos uploaded (JPEG, PNG, HEIC) are processed server-side before being stored and served to devices. The goal is to make any photo look good on the B&W e-ink display.

Input
Any photo format
JPEG · PNG · HEIC · up to 20MB
resize
Step 1 · Sharp.js
Resize to 400×300
Cover crop · centred · no stretching
grayscale
Step 2 · Sharp.js
Convert to grayscale + normalise contrast
Boosts apparent detail on e-ink
dither
Step 3 · Floyd-Steinberg
1-bit dithering
Error diffusion → creates illusion of depth with just black and white pixels
encode
Output
1-bit BMP, 400×300
~15KB · served directly to ESP32 · no decompression needed
Floyd-Steinberg dithering is the same algorithm used by the original Macintosh, early digital cameras, and e-reader manufacturers. It spreads quantisation error to neighbouring pixels, making B&W photos look remarkably detailed on constrained displays.

Future ideas & v2 scope

The POC is deliberately scoped small. Here's the backlog of everything considered, deferred, or imagined.

v2 · Near term
MEMRY Snap (NFC)
Battery-free NFC variant. Needs proper iOS CoreNFC testing and RF antenna tuning. Slimmer (7mm). Complementary product.
v2 · Near term
Flutter app
Single native app for both iOS and Android. Required for Snap NFC tap. Optional for Live — web is sufficient.
🤖 v2 · AI — hardware untouched
LLM natural language commands
Web app understands: "Show something cheerful on Jo's fridge" or "Put a beach photo from last summer." LLM picks from photo library, publishes automatically. Pure software — zero hardware change.
📷 v2 · AI — hardware untouched
Google Photos integration
Connect your Google Photos account. Set automated rules: "Every Monday, pick a photo from the past year." LLM + vision model picks the most heartfelt image. MEMRY just displays as normal.
🗓️ v2 · AI — hardware untouched
Smart scheduling in natural language
"Show Jo's birthday photos this week" or "Rotate holiday pics for December." LLM parses intent, creates a rotation schedule. Vision model scores photos for warmth, faces, and composition.
v2 · Near term
Push updates
WebSocket/MQTT instead of polling. Photo appears within seconds of publishing. Needs always-on WiFi listener — more battery. Worth it for "instant magic".
v2 · Near term
Image filters
Vintage film, high contrast, bright/dark adjustments. Server-side at upload time. Could also let users preview different dither styles.
v2 · Near term
Caption on strip
Render caption text into the bottom white strip of the Polaroid (part of the BMP). Users add caption at upload time.
v3 · Medium
Custom PCB
Replace XIAO devkit with custom PCB. Remove USB headers, LEDs, excess pins. Get to 8mm total shell thickness. MOQ ~50pcs from JLCPCB.
v3 · Medium
Injection moulded shell
Replace 3D printed PLA. MOQ ~500 units, ~₹150/unit vs ₹600. Required to make retail economics work. Colours: cream, black, sage.
v3 · Medium
Public launch
Anyone can buy a MEMRY. Gift-ready packaged units. QR code inside box links to pairing flow. Subscription for contributor management.
idea · Wild
MEMRY Wall
2×2 or 3×3 grid of MEMRY units that tile into one large e-ink display. Each refreshes independently. Together they show one big photo.
idea · Wild
Reactions
Web app shows emoji reaction overlay on the Polaroid when friends react to a photo. Like Instagram stories, but on your fridge.
idea · Wild
MEMRY for teams
Office walls, conference rooms. Device shows rotating team photos, announcements, metrics. Enterprise tier with admin controls.
idea · Wild
Color display
6-color e-ink (ACeP) when power efficiency improves enough. VidaBay CEO confirmed 6-color still needs battery. Watch ESL market — they'll crack it first.
